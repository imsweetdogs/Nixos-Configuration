# 🧩 Модули системы

Модули в этой конфигурации позволяют капсулировать отдельные функции системы так же,
как это делает официальный *NixOS Modules System*, но с дополнительной структурой:
архитектура → область (system / desktop / services / и т.  e) → модуль.

```
config/
└── modules/
    ├── <архитектура>/          # x86_64-linux, aarch64-linux …
    │   └── system/
    │       └── boot/
    │           └── default.nix  # сам модуль
    └── default.nix             # сканер, автоматически импортирующий всё выше
```

Такой подход упрощает поддержку нескольких архитектур (например, десктоп + сервер на ARM) и
помогает отделять специфичные параметры.

### Как добавить новый модуль

1. Создайте каталог согласно схеме `<архитектура>/<категория>/<имя_модуля>/`.
2. Поместите в него `default.nix` с описанием опций и `config`-секцией.
3. При необходимости добавьте страницу в `docs/modules/...` и ссылку в таблицу выше.
