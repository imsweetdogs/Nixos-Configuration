# 👤 Пользователи в конфигурации

В конфигурации **Sweetdogs NixOS** каждый пользователь описывается отдельным Nix-модулем. Это позволяет

1. Чётко разделять настройки разных людей.
2. Переиспользовать и расширять конфигурации без конфликтов.

---

## 📂 Где лежат пользовательские модули?

```
config/
└── users/
    ├── default.nix          # агрегатор, подключающий выбранных пользователей
    └── <username>/
        └── default.nix      # личный модуль конкретного пользователя
```

*Каждая папка `<username>` названа так же, как UNIX-аккаунт.*

### Агрегатор `config/users/default.nix`

```nix
{ flake, ... }: {
  imports = map (user: "${flake.conf.path}/users/${user}") flake.conf.system.users;
}
```

- Список пользователей берётся из `flake.conf.system.users` (см. `config/default.nix`).
- Для каждого имени формируется путь и импортируется соответствующий модуль.

### Пример модуля пользователя

```nix
# config/users/sweetdogs/default.nix
{ flake, lib, ... }: {
  users.users.sweetdogs = {
    description   = "Aleksey Baev";
    createHome    = true;       # создать домашний каталог
    isNormalUser  = true;       # обычный пользователь (не system-account)
    hashedPassword = flake.conf.system.baseHashedPassword;
    extraGroups    = [ "networkmanager" "wheel" "input" "video" "libvirtd" "adbusers" ];
  };
}
```

> На практике в этот модуль имеет смысл добавлять ПО, которое должно быть установлено специально для данного пользователя, когда использование **Home Manager** выглядело бы избыточным (для него существует отдельная структура модулей, также разделённая по архитектурам). Чаще всего его применяют для установки общего софта, присутствующего на всех архитектурах, и назначения пользователю нужных групп доступа.

## 👥 Текущие пользователи

| Имя | Описание | Доп. группы |
|-----|----------|-------------|
| `sweetdogs` | Aleksey Baev (основной пользователь) | `networkmanager`, `wheel`, `input`, `video`, `libvirtd`, `adbusers` |

Чтобы добавить нового пользователя:

1. Укажите его имя в списке `system.users` в файле `config/default.nix`.
2. Создайте каталог `config/users/<имя>` и файл `default.nix` с нужными настройками.
