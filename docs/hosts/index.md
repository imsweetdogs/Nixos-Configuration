# 🖥️ Хосты

В этой конфигурации каждый хост (то есть конечная NixOS-система) описывается
Nix-файлом внутри каталога `config/hosts`.  Структура каталога построена по
принципу *архитектура → тип хоста*.

```
config/hosts/
├── x86_64-linux/          # архитектура
│   ├── common/            # "тип" или конкретный хост
│   │   ├── default.nix    # описание хоста (hostname, modules)
│   │   └── system.nix     # дополнительные параметры системы
│   └── ...
└── default.nix            # генератор флейк-выходов `nixosConfigurations`
```

---

## Как работает генератор (`config/hosts/default.nix`)

Файл `hosts/default.nix` экспортирует атрибутный набор, который превращается в
`flake.outputs.nixosConfigurations`.

1. Функция `mkHost` создаёт систему через `inputs.nixpkgs.lib.nixosSystem`,
   передавая:
   * путь к оверлеям `nixpkgs`,
   * модуль `inputs.disko.nixosModules.disko`,
   * модуль, задающий `networking.hostName`,
   * пользовательские модули из `modules` в описании хоста.
2. Скрипт рекурсивно проходит по подкаталогам архитектур и собирает все файлы
   `default.nix`, возвращая список `{ name, value }`.
3. Список конвертируется в attrset функцией `builtins.listToAttrs`.

Таким образом любой файл `hosts/<arch>/<name>/default.nix`, который возвращает
запись `{ hostname, modules }`, автоматически становится конфигурацией `flake`-а
под ключом `<name>`.

## Пример минимального хоста

```nix
# config/hosts/x86_64-linux/desktop/default.nix
{ inputs, flake, ... }: {
  hostname = "sweet-desktop";
  modules = [
    # Подключаем нужные системные модули
    "${flake.conf.path}/users"                # все пользователи
    ./system.nix   # набор системных настроек
  ];
}
```

После добавления файла команда

```bash
sudo nixos-rebuild switch --flake .#sweet-desktop
```

соберёт и установит систему `sweet-desktop`.

## Совет по структуре

Если у вас несколько хостов с похожими настройками, создайте общий каталог
(например, `common/`), где опишите базовые модули, а в конкретных хостах
переиспользуйте этот набор через `import ./../common` или `modules = [ ../common ]`.
